🔴 CRITICAL ISSUES (Fix immediately):

1. STALE DATA BUG - intraday.py
   └─ Variable `_last_ts` undefined, checked but never initialized
   └─ Impact: May process same candle 100x, generate duplicate alerts
   └─ Severity: HIGH - affects alert accuracy
   └─ Fix complexity: LOW (1 line)
   └─ Time to fix: 5 minutes

2. THREAD-UNSAFE GLOBALS - daily_builder.py
   └─ Shared classifier_state dictionary accessed without locks
   └─ Impact: Race condition if two threads call classify_stock() simultaneously
   └─ Severity: HIGH - silent data corruption
   └─ Fix complexity: MEDIUM (add threading.Lock())
   └─ Time to fix: 15 minutes

3. CHECKPOINT FILES ON EPHEMERAL STORAGE
   └─ Saved to /tmp/ or Railway ephemeral filesystem
   └─ Impact: Lost on restart/redeploy, no persistent audit trail
   └─ Severity: HIGH - data recovery impossible
   └─ Fix complexity: MEDIUM (move to persistent DB or EFS)
   └─ Time to fix: 30 minutes

4. PRICE CACHE TTL MISMATCH - config.py
   └─ Cache TTL: 180 seconds, but 15m scanner runs every 5 minutes
   └─ Impact: Cache miss every 5min cycle, API rate limits hit harder
   └─ Severity: MEDIUM - affects API quota efficiency
   └─ Fix complexity: LOW (change 180→60)
   └─ Time to fix: 5 minutes

⚠️ MEDIUM SEVERITY ISSUES (Fix before production):

5. DATABASE CONNECTION POOL EXHAUSTION
   └─ Max 30 concurrent connections
   └─ No timeout on failed connections
   └─ Risk: If Postgres down, 30+ connections hang for 30s
   └─ Fix: Add connection timeout + circuit breaker

6. TELEGRAM RATE LIMITING
   └─ 30 alerts/second limit, system sends in batches of 10
   └─ No queue with backpressure
   └─ Risk: Bulk alert spike could lose some alerts
   └─ Fix: Implement persistent alert queue, batch processing

7. NSE DELIVERY DATA TIMING
   └─ Published at 6:00 PM, EOD scanner runs at 6:30 PM
   └─ Risk: May use stale/incomplete data
   └─ Mitigation: Already has fallback to previous day
   └─ Acceptable: Yes, with monitoring

8. YFINANCE RATE LIMITS
   └─ System makes ~706 API calls/day, near limit
   └─ No fallback if yfinance blocked
   └─ Risk: Single ISP block = system failure
   └─ Fix: Add fallback to AlphaVantage or Alternative provider
