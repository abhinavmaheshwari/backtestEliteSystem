───────────────────────────────────────────────────────────────────────────────
                           WORKFLOW OVERVIEW
───────────────────────────────────────────────────────────────────────────────

Step 1: Bayesian Proposes
   └─ submit_bayesian_update_for_approval()
      └─ Saved to: bayesian_model_updates (PENDING status)
      └─ NOT in: score_weight_log (live weights table)
      └─ Result: Proposal visible but NOT USED

Step 2: Admin Reviews
   └─ get_pending_bayesian_updates() fetches from DB
   └─ Admin sees: old weights vs new weights, win rate, trades analyzed
   └─ Admin decides: [APPROVE] or [REJECT]

Step 3a: Admin Approves
   └─ approve_bayesian_update()
      └─ INSERT into score_weight_log (NOW LIVE!)
      └─ UPDATE bayesian_model_updates status=APPROVED
      └─ Record: who approved, when, comment
      └─ Result: Weights now LIVE

   OR

Step 3b: Admin Rejects
   └─ reject_bayesian_update()
      └─ Update status=REJECTED
      └─ score_weight_log UNCHANGED
      └─ Result: Current weights stay live

Step 4: Scanners Use Live Weights
   └─ get_current_bayesian_model()
      └─ Reads ONLY from score_weight_log (approved weights)
      └─ Returns: v2 (if approved) or v1 (if rejected)
      └─ Result: Using ONLY what admin approved

───────────────────────────────────────────────────────────────────────────────
                      TABLE SEPARATION (ISOLATION)
───────────────────────────────────────────────────────────────────────────────

bayesian_model_updates Table:
├─ Holds: PENDING, APPROVED, REJECTED proposals
├─ Status: Not read by any scanner
├─ Used by: Admin dashboard, Bayesian updater
└─ Effect: PENDING weights never reach scanners ❌

score_weight_log Table:
├─ Holds: LIVE weights ONLY (approved entries)
├─ Status: Read by get_current_bayesian_model()
├─ Used by: All scanners for calculations
└─ Effect: Only approved weights used for scoring ✅

get_current_bayesian_model():
├─ Reads from: score_weight_log (approved ONLY)
├─ Never touches: bayesian_model_updates (PENDING table)
├─ Returns: {BULL: {version: v2, weights: {...}}}
└─ Guarantee: Cannot return PENDING weights ✅

───────────────────────────────────────────────────────────────────────────────
                          CODE ISOLATION PROOF
───────────────────────────────────────────────────────────────────────────────

Bayesian Updater Path:
   get_current_bayesian_model() [reads approved]
      ↓ Analyze trades
      ↓ Propose weights
   submit_bayesian_update_for_approval() [writes PENDING]
   ❌ NEVER writes to score_weight_log
   ❌ NEVER affects live calculations
   ❌ NEVER called by scanners

Admin Approval Path:
   approve_bayesian_update() [ONLY inserts to live table]
      ├─ INSERT into score_weight_log (NOW LIVE)
      └─ UPDATE bayesian_model_updates to APPROVED

Scanner Path:
   get_current_bayesian_model() [reads live only]
      ↓ Call calculate_score()
      ↓ Save alert with model_version
   ❌ NEVER touches bayesian_model_updates
   ❌ NEVER sees PENDING weights

Guarantee: ✅ Three completely separate code paths!

   Purpose: List pending for admin review
